 时间复杂度分析：代码执行的时间
int cal(int n) {
    int sum = 0;
    int i = 0;
    for ( ; i<n; i++){
        sum += i;
    }
    return sum;
}
假设一行代码执行的时间为一个单位（unit_time）,
第二三行执行时间为 2*unit_time,第四五行执行时间为 2*n*unit_time
总的执行时间 T(n) = (2+2*n)*unit_time, 
得出代码的执行时间和 代码的执行次数n成正比,总结得出
T(n) = O(f(n)) 
n表示数据规模的大小，f(n) 表示每行代码执行的次数总和，这就是大 O 时间复杂度表示法
大 O 时间复杂度实际上并不具体表示代码真正的执行时间，表示代码执行时间随数据规模增长的变化趋势

常用的复杂度
常数阶 O(1)
对数阶 O(logn)
线性阶 O(n)
线性对数 O(nlogn)
平方阶  O(n^2), 立方 O(n^3),k次方O(n^k)
指数阶  O(2^n)
阶乘     O(n!)
